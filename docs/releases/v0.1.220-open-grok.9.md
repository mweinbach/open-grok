# Open Grok v0.1.220-open-grok.9

Open Grok is Grok Build with ChatGPT Codex optimizations, packaged as the
independent `open-grok` command with isolated state under `OPENGROK_HOME`.

## Upstream reliability sync

- Path normalization, validated HTTP hook redirect handling, and managed-policy
  synchronization now include the latest compatible upstream hardening.
- Stdio updates are restricted to managed installs, abandoned permission
  requests are canceled, and workspace session bindings are released promptly.
- Session listing moves JSONL parsing off the async executor, and permission
  prompts retain the decision context used by follow-up classification.
- Dependency-backed test support is available across sibling pager crates.

## Authentication and enterprise voice

- Workspace OAuth requests the scopes needed by the current Grok workspace
  contract.
- First-party credentials supplied by an external auth provider can participate
  in session-only model access and refresh recovery without changing endpoint
  trust decisions.
- Tools and voice fall back to `XAI_API_KEY` or the persisted xAI API key when
  no usable session bearer is available, while respecting API-key policy pins.
- Voice inherits the resolved enterprise xAI API endpoint, converts secure HTTP
  bases to WSS, de-duplicates `/v1`, and refuses plaintext bearer transport.

## Navigation, settings, and terminal interaction

- `/jump` opens a fast transcript destination picker, while `/timeline` toggles
  a persistent activity rail for navigating the session.
- The settings modal is split into focused state, input, rendering, and test
  modules while preserving Codex, Kimi, Code Mode, recap, memory, and secondary
  model controls from the fork.
- `/minimal` and `/fullscreen` are session-scoped switches with clear return
  cues, and minimal mode now surfaces folder-trust state without losing provider
  routing.
- Adjacent edit blocks coalesce without hiding meaningful gaps, and semantic
  file links preserve exact targets across VS Code SSH delegation and browser
  fallback.

## Headless and PTY reliability

- Headless turns briefly drain late ACP notifications before exit so final text
  and background-task lifecycle events are not dropped across channel races.
- PTY reads now distinguish timeout from process exit, provide stable condition
  waits and final-output draining, and run in smaller scheduling families for
  more deterministic integration coverage.

## Install

This release provides a raw binary for **Apple Silicon macOS only**.

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/download/v0.1.220-open-grok.9/install.sh | bash
open-grok --version
```

The installer verifies the published SHA-256 digest and atomically installs
only `open-grok` to `${OPENGROK_HOME:-$HOME/.opengrok}/bin`. It does not replace
or create `grok` or `agent` commands, and it does not read or write `~/.grok`.
