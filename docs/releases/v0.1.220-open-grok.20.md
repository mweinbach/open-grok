# Open Grok v0.1.220-open-grok.20

Open Grok is Grok Build with ChatGPT Codex, Kimi, and Fireworks AI provider
support, packaged as the independent `open-grok` command with isolated state
under `OPENGROK_HOME`.

This release adds the `workflow` tool: deterministic multi-subagent
orchestration driven by a JavaScript script, in the spirit of Claude Code's
Workflow harness. Where `agent_swarm` fans one prompt template over a list of
items, a workflow script owns the control flow — multi-stage pipelines,
adversarial verification passes, judge panels, loop-until-done discovery —
while the host owns concurrency, journaling, progress, and cancellation.

## Workflow orchestration

- Scripts begin with a pure-literal `export const meta = { name, description,
  phases? }` header (validated without executing JavaScript) and run in the
  code-mode V8 runtime with top-level `await` and a `return` value that
  becomes part of the tool result.
- Script hooks: `agent(prompt, opts)` spawns a real foreground subagent and
  resolves to its final text (`opts`: `label`, `phase`, `schema`, `model`,
  `effort`, `isolation: "worktree"`, `agentType`); `parallel(thunks)` runs
  tasks concurrently; `pipeline(items, ...stages)` runs items through stages
  with no inter-stage barrier; `phase()` / `log()` drive live progress;
  `args`, `meta`, and `budget` expose the call input, parsed header, and
  token accounting.
- `schema` enforces a JSON output contract: the child's reply is parsed (bare,
  fenced, or embedded JSON) with one corrective resume retry before the call
  reports failure. Failed or cancelled agents resolve to `null` instead of
  aborting the script.
- Runs are journaled under the session directory; calling `workflow` again
  with `resume_from_run_id` replays unchanged `agent()` calls instantly and
  re-runs only what changed. `Date.now()`, `Math.random()`, and argless
  `new Date()` throw inside scripts so replays stay deterministic.
- Concurrency is capped (override `OPENGROK_WORKFLOW_MAX_CONCURRENCY`;
  per-agent timeout honors `OPENGROK_SUBAGENT_TIMEOUT_MS`), runs are limited
  to 1000 agents, and children carry the parent prompt id so cancelling the
  turn cancels the whole cohort. Workflow children render as a grouped cohort
  card with live progress, and the workflow tool card streams phase and log
  lines while the script runs.

## Flat-tree hardening

- The depth-1 toolset strip is now a single shared helper that removes every
  nested spawn surface — `task`, `agent_swarm`, and the new `workflow` — from
  child toolsets, with direct unit coverage. Children cannot orchestrate
  further children, no matter which orchestrator launched them.
- No subagent capability mode re-grants orchestration tools, `workflow` is
  batch-exclusive like `agent_swarm` (rejected when combined with other tool
  calls, with per-call error results), and it stays a top-level direct-only
  control in Code Mode Only rather than appearing under `tools.*`.

## Upgrade

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/latest/download/install.sh | bash
```

Or from a running session: `/update`.
