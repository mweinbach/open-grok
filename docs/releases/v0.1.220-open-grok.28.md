# Open Grok v0.1.220-open-grok.28

Open Grok is Grok Build with ChatGPT Codex, Kimi, and Fireworks AI provider
support, packaged as the independent `open-grok` command with isolated state
under `OPENGROK_HOME`.

This release syncs the upstream `xai-org/grok-build` snapshot `a5727c59`
(upstream v0.2.110) and makes Antigravity subagents fully capable by
default.

## Antigravity subagents: full access by default

Headless `agy` auto-denies write and command tools without its
skip-permissions flag, which surfaced as constant permission errors for
implementation-stage subagents. Antigravity members now run **with the
flag by default**. Two guardrails remain: spawns whose capability mode is
pinned read-only (review/audit stages) never get the flag, and you can
opt out globally with `[antigravity] skip_permissions = false` — now also
exposed in `/settings` as **"Antigravity full access"** (restart
required). Note this trusts agy's agent with workspace writes and command
execution.

## From the upstream sync

- **Queued prompts** — upstream's fix for the combine-queued edit-hold
  race (Open Grok ships `combine_queued_prompts` on).
- **Security** — `Bash(git:*)` allowlist rules now prefix-match the whole
  command chain; auto-mode's classifier handles fail-closed gates,
  honors recorded approvals for repeat actions, and prompts on timeout
  instead of silently denying.
- **Runaway protection** — turns that issue the exact same tool call 16
  times in a row are stopped.
- **Tasks and subagents** — background shells report real exit codes;
  subagent completion drains are scoped to the owning session.
- **Doctor** — remediation consolidated, fixes applicable from the TUI,
  startup warnings routed into doctor. The ssh-wrap fix now writes
  fork-correct aliases (`open-grok wrap ssh` — previously it wrote a
  `grok` alias that doesn't exist on Open Grok installs).
- **Sessions** — compaction checkpoint files are copied when forking,
  a one-shot occurrence journal persists, relocation recovery is
  integrated, and Esc cancels the running turn in non-vim/minimal modes.
- **Tool overrides** — upstream's `toolOverrides` wire mechanism is in
  (client-side control of hosted web/X-search options), integrated with
  the fork's code-mode client tools.
- **Upstream now enables workflows by default** — validating the `.27`
  divergence; Open Grok additionally keeps ignoring the remote kill
  switch.
- Marketplace hardening (git-source timeouts, remove-by-name, unbrick),
  out-of-process macOS mic capture, Ctrl+Z in shortcuts help, dashboard
  hit-area fixes, and the npm binary installing under the managed home.

## Fork notes

- Remote feature flags stay ignored wholesale (`.27` behavior), so
  upstream's new remote-gated surfaces — the coding-data upsell banner
  and the privacy-notice rollout — are permanently inert here.
- Upstream's new `voice_mode` remote/requirements gating was not adopted;
  Open Grok's voice behavior is unchanged.
- The fork's latent rich web-search config (never reachable from any
  session surface) was dropped in favor of upstream's tool-overrides
  model.
- The e2e test harness moved to upstream's `TestSandbox` architecture,
  rebranded for `OPENGROK_HOME`/`.opengrok`; the fork's Codex OAuth and
  auxiliary-routing e2e tests were ported onto it.
