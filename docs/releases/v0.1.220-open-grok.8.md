# Open Grok v0.1.220-open-grok.8

Open Grok is Grok Build with ChatGPT Codex optimizations, packaged as the
independent `open-grok` command with isolated state under `OPENGROK_HOME`.

## Provider-aware login

- `/login` now opens a provider picker for xAI Grok, ChatGPT Codex, and Kimi.
- The picker works in active agent sessions, attached dashboard sessions, and
  the session-less dashboard.
- Provider aliases such as `/login xai`, `/login codex`, and `/login kimi`
  continue directly into the corresponding authentication flow.
- Kimi opens a focused settings-style API-key editor and immediately queries
  the provider's model catalog after the key is saved.

## Kimi coding support

- Kimi is a first-class API-key provider using OpenAI-compatible Chat
  Completions and Bearer authentication.
- Kimi model discovery merges into the provider-partitioned catalog without
  displacing xAI or Codex models.
- The built-in `kimi-k3` fallback remains available when live model discovery
  is unavailable, and `MOONSHOT_API_KEY` remains supported as an environment
  override.
- Open Grok does not invent provider-hosted Kimi tools when the platform does
  not advertise them.

## Generalized response foundation

- Provider profiles define wire format, authentication, hosted-tool dialect,
  header policy, and access to provider-specific services.
- Sampling routes through provider adapters, keeping Codex Responses, xAI
  formats, and Kimi Chat Completions behind a shared extension boundary.
- API-key and OAuth credentials resolve per provider and endpoint so new
  providers can be added without inheriting unrelated authentication state.

## Credential and modal hardening

- Kimi credentials are restricted to trusted Kimi and Moonshot HTTPS hosts.
- Secret input is masked, Debug-redacted, zeroized on drop, excluded from input
  recording, and protected across fragmented terminal paste events.
- Login and secret modals retain keyboard priority over dashboard and overlay
  shortcuts, preventing a credential draft from being hidden or leaked into
  ordinary terminal input.

## Install

This release provides a raw binary for **Apple Silicon macOS only**.

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/download/v0.1.220-open-grok.8/install.sh | bash
open-grok --version
```

The installer verifies the published SHA-256 digest and atomically installs
only `open-grok` to `${OPENGROK_HOME:-$HOME/.opengrok}/bin`. It does not replace
or create `grok` or `agent` commands, and it does not read or write `~/.grok`.
