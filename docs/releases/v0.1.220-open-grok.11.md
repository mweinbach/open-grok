# Open Grok v0.1.220-open-grok.11

Open Grok is Grok Build with ChatGPT Codex optimizations, packaged as the
independent `open-grok` command with isolated state under `OPENGROK_HOME`.

## Provider-aware Code Mode

- xAI Responses Code Mode now exposes `exec` as a normal function whose
  required `source` field contains raw JavaScript. It no longer sends OpenAI's
  unsupported native `custom` tool type to `cli-chat-proxy.grok.com`.
- Mixed **Code Mode** keeps ordinary tools top-level alongside `exec` and
  `wait`, which is the normal xAI configuration. Explicit **Code Mode Only**
  keeps ordinary tools behind JavaScript `tools.*` calls.
- Settings now offers **Direct**, **Code Mode**, and **Code Mode Only** as
  separate choices. Legacy booleans remain readable, while new configuration
  writes use the explicit string values.
- OpenAI Codex models that require Code Mode Only continue to override the user
  preference. Codex retains its native custom/freeform raw-JavaScript `exec`
  contract; incompatible routes fail before a request is sent.
- One provider-aware tool-surface policy now drives turns, compaction, forks,
  resume, and context accounting. Responses requests are projected exactly
  once, with pre-network guards preventing unsupported custom content from
  reaching xAI.

## Session and runtime reliability

- The persistent V8 runtime is generation-bound and replaced after rewind or
  incompatible provider/tool-mode transitions. Stale callbacks and yielded
  cell identifiers fail closed instead of attaching to a replacement timeline.
- Model and provider changes serialize with active turns, preserve FIFO prompt
  ordering, and stage replacement state before resetting the runtime.
- Conversation and file rewind now use staged, compensating transactions so a
  later failure restores already-applied files instead of leaving history and
  the workspace out of sync.
- Resolved Code Mode policy persists across cold resume, while provider-changing
  forks receive a fresh provider-local surface and credentials.
- xAI preflight and post-401 token recovery now share the same stable session,
  BYOK, and first-party-host gate. Stale session classifications heal on the
  canonical xAI route, while custom endpoints can never receive the built-in
  session token.
- Provider-changing model overrides keep provider-native reasoning-summary
  defaults and honor explicit support/detail overrides without inheriting a
  donor provider's capabilities.
- Codex Responses accepts actionless hosted web-search progress events without
  aborting an otherwise valid stream.

## Install

This release provides a raw binary for **Apple Silicon macOS only**.

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/download/v0.1.220-open-grok.11/install.sh | bash
open-grok --version
```

The installer verifies the published SHA-256 digest and atomically installs
only `open-grok` to `${OPENGROK_HOME:-$HOME/.opengrok}/bin`. It does not replace
or create `grok` or `agent` commands, and it does not read or write `~/.grok`.
