# Open Grok v0.1.220-open-grok.2

Open Grok is Grok Build with ChatGPT Codex optimizations, packaged as the
independent `open-grok` command.

## Highlights

- Adds a first-launch TUI choice between ChatGPT Codex and xAI Grok. Each path
  validates only its own credentials, and Codex can start without xAI access.
- Loads OpenAI model metadata from the same authenticated Codex
  `/models?client_version=...` contract used by codex-rs. Live names, context
  windows, reasoning levels, search support, and tool modes override the
  embedded Codex fallback catalog.
- Uses each live model's `auto_compact_token_limit` and `comp_hash`, and ports
  Codex automatic/manual compaction through `POST /responses/compact` with
  exact opaque replacement-item replay.
- Preserves Codex's per-turn `x-codex-turn-state` across tool continuations,
  retries, client rebuilds, and compaction without carrying it into another
  prompt or xAI request.
- Keeps Max and Ultra distinct in model settings while encoding both as Codex
  `max`; selecting Ultra on a live model that declares multi-agent v2 receives
  codex-rs's proactive request policy.
- Keeps the live Codex catalog in a five-minute, account-scoped cache at
  `$OPENGROK_HOME/codex_models_cache.json`, separate from the xAI model cache.
- Shows OpenAI and xAI models together in `/model`, including provider and
  context-window labels, and keeps provider-specific access controls correct
  when switching models. Existing conversations rebuild the provider harness
  in place instead of requiring a new session.
- Preserves Codex Code Mode Only behavior for GPT-5.6 Sol and routes hosted web
  search through the selected provider's native API. The TUI shows decoded
  inner tools while hiding raw `exec`/`wait` transport details live and during
  replay. xAI's current `x_search_call` response shape renders as a normal
  provider-hosted X Search lifecycle.
- Keeps subagents, task waiting, schedulers, monitors, goals, plan mode, and
  user questions available in the Codex harness as well as the Grok harness.
- Shows Codex quota windows beside xAI usage in `/usage`, with independent
  failure handling.
- Keeps all runtime state under `${OPENGROK_HOME:-$HOME/.opengrok}` and stores
  xAI and Codex credentials separately, so Open Grok can coexist with Grok
  Build.

## Install

This release provides a raw binary for **Apple Silicon macOS only**.

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/download/v0.1.220-open-grok.2/install.sh | bash
open-grok --version
```

The installer verifies the published SHA-256 digest and atomically installs only
`open-grok` to `${OPENGROK_HOME:-$HOME/.opengrok}/bin`, unless
`OPEN_GROK_BIN_DIR` is set. It does not replace or create `grok` or `agent`
commands.

## Compatibility and provenance

Open Grok does not fall back to upstream Grok Build state. Credentials,
settings, sessions, caches, skills, plugins, and project configuration remain
isolated under `.opengrok` / `OPENGROK_HOME`.

Open Grok is a community fork of Grok Build and is not affiliated with,
sponsored by, or endorsed by xAI, SpaceXAI, or OpenAI. Codex behavior is ported
from pinned revisions of [`openai/codex`](https://github.com/openai/codex); see
[`docs/code-mode-port.md`](https://github.com/mweinbach/open-grok/blob/v0.1.220-open-grok.2/docs/code-mode-port.md),
[`docs/codex-provider-port.md`](https://github.com/mweinbach/open-grok/blob/v0.1.220-open-grok.2/docs/codex-provider-port.md),
