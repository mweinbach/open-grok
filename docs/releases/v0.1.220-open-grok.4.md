# Open Grok v0.1.220-open-grok.4

Open Grok is Grok Build with ChatGPT Codex optimizations, packaged as the
independent `open-grok` command.

## Codex parity and fixes

- Matches codex-rs HTTP cache continuity: every Codex Responses request uses a
  stable session-scoped `prompt_cache_key`, while full-input HTTP requests do
  not send the WebSocket-only `previous_response_id` field.
- Accepts the forward-compatible `response.metadata` stream event, preserves
  the first `x-codex-turn-state` value across one logical turn, and safely
  observes response IDs without treating them as HTTP conversation state.
- Retains completed `response.output_item.done` entries when the terminal event
  omits output, including encrypted reasoning, messages, and tool calls.
- Passes through streamed reasoning summaries and follows the live Codex model
  catalog's summary capability and default. The embedded GPT-5.4, GPT-5.5, and
  GPT-5.6 fallbacks omit summaries unless live metadata enables them.
- Enables Codex Remote Compaction V2 by default. It uses the normal streaming
  `/responses` endpoint, advertises `remote_compaction_v2`, appends the native
  compaction trigger, retains a bounded newest real-user tail, and replays the
  opaque encrypted compaction item exactly. Set
  `[features] remote_compaction_v2 = false` or
  `OPENGROK_REMOTE_COMPACTION_V2=0` to select the legacy
  `/responses/compact` protocol.
- Keeps compaction response IDs and token usage in diagnostics and the model
  call ledger without overwriting the most recent visible turn's context
  accounting.

These Codex-specific projections do not change xAI request bodies, xAI search,
Grok compaction, or Grok model behavior.

## Updates

The release updater now downloads the public GitHub release, verifies the
published SHA-256 digest and binary version, installs it under the isolated
`OPENGROK_HOME`, and atomically advances the managed `open-grok` command. The
running process is never overwritten in place.

Version `0.1.220-open-grok.3` contained the older no-op updater, so install this
release once with the command below. Starting with this release, future updates
can be installed with `open-grok update`, and automatic launch checks can
activate a verified download for restart.

## Install

This release provides a raw binary for **Apple Silicon macOS only**.

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/download/v0.1.220-open-grok.4/install.sh | bash
open-grok --version
```

The installer verifies the published SHA-256 digest and atomically installs only
`open-grok` to `${OPENGROK_HOME:-$HOME/.opengrok}/bin`, unless
`OPEN_GROK_BIN_DIR` is set. It does not replace or create `grok` or `agent`
commands. Credentials, settings, sessions, and caches remain separate from
`~/.grok`.

## Compatibility and provenance

Open Grok does not fall back to upstream Grok Build state. Credentials,
settings, sessions, caches, skills, plugins, and project configuration remain
isolated under `.opengrok` / `OPENGROK_HOME`.
