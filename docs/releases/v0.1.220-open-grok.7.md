# Open Grok v0.1.220-open-grok.7

Open Grok is Grok Build with ChatGPT Codex optimizations, packaged as the
independent `open-grok` command with isolated state under `OPENGROK_HOME`.

## Kimi coding models

- Kimi is now a first-class API-key provider using OpenAI-compatible Chat
  Completions and Bearer authentication.
- **Settings → Models → Kimi API key** accepts a masked credential, stores it in
  the provider-scoped owner-only credential store, and queries Kimi's live
  `/v1/models` catalog.
- The built-in `kimi-k3` fallback is available when live discovery is
  unavailable. `MOONSHOT_API_KEY` remains supported as an environment override.
- Kimi receives standard client-side function tools. Open Grok does not invent
  or enable provider-hosted Kimi tools that the platform does not advertise.
- Active Kimi sessions rebind after a key change without requiring an app
  restart.

## Generalized provider foundation

- Provider profiles now define supported wire formats, authentication style,
  hosted-tool dialect, header policy, and access to xAI-only services.
- Sampling requests are routed through provider adapters, keeping Codex
  Responses, xAI formats, and Kimi Chat Completions isolated behind one shared
  extension boundary.
- Live model catalogs merge by provider partition, preventing one provider's
  records or defaults from displacing another provider's models.
- API-key and OAuth credentials are resolved per provider and endpoint. Custom
  endpoints cannot inherit unrelated xAI or ChatGPT session credentials.

## Credential and request hardening

- Kimi credentials are restricted to trusted Kimi and Moonshot HTTPS hosts;
  request logs and model-query errors redact credential material.
- Credential updates use a shared advisory lock so xAI and Kimi writes cannot
  overwrite sibling scopes. Failed reads or deletes are surfaced instead of
  being reported as successful clears.
- The settings secret buffer is Debug-redacted, zeroized on drop, excluded from
  generic settings persistence and input recording, and safely handles
  fragmented terminal paste events.
- Kimi requests omit model-owned sampling and penalty fields, do not receive
  xAI-only headers or hosted tools, and do not enter xAI OAuth refresh paths.

This release includes all provider-aware recap and memory routing, Codex
reasoning-summary defaults, mapped Code Mode tool presentation, OAuth,
compaction, updater, and `.opengrok` isolation work from
`v0.1.220-open-grok.6`.

## Install

This release provides a raw binary for **Apple Silicon macOS only**.

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/download/v0.1.220-open-grok.7/install.sh | bash
open-grok --version
```

The installer verifies the published SHA-256 digest and atomically installs
only `open-grok` to `${OPENGROK_HOME:-$HOME/.opengrok}/bin`. It does not replace
or create `grok` or `agent` commands, and it does not read or write `~/.grok`.
