# Open Grok v0.1.220-open-grok.21

Open Grok is Grok Build with ChatGPT Codex, Kimi, and Fireworks AI provider
support, packaged as the independent `open-grok` command with isolated state
under `OPENGROK_HOME`.

This patch release fixes two defects found in the first production workflow
run on v0.1.220-open-grok.20.

## Fireworks: replayed history no longer 400s

Fireworks validates the chat schema strictly and rejected any request whose
replayed assistant message still carried Open Grok's internal per-message
`model_id` attribution ("Extra inputs are not permitted"). Every
multi-request Fireworks turn hit this — a child agent died as soon as it made
its second model call, which broke GLM workflow agents, GLM `task` subagents,
and multi-turn GLM chats alike. The Fireworks adapter now strips the internal
bookkeeping field from every outbound message; the request-level `model`
field remains the selector, and other providers' wire behavior is unchanged.

## Workflow: resume chains are no longer lossy

A resumed workflow run now re-journals the results it replays, so resuming
from an intermediate resumed run serves its full set from the journal instead
of re-running agents that run had itself replayed.

## Upgrade

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/latest/download/install.sh | bash
```

Or from a running session: `/update`.
