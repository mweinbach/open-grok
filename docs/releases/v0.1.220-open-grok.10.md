# Open Grok v0.1.220-open-grok.10

Open Grok is Grok Build with ChatGPT Codex optimizations, packaged as the
independent `open-grok` command with isolated state under `OPENGROK_HOME`.

## Codex image preparation

- Codex Responses requests now prepare inline images before the wire body is
  built, matching codex-rs `image_preparation::prepare_response_items`.
- Invalid `data:` base64 payloads (including unpadded or whitespace-tainted
  values), remote `http(s)://` image URLs, and tool-output `detail: low` are
  replaced with short text placeholders so a single bad tool image cannot brick
  the turn with
  `Invalid 'input[N].output[M].image_url' … invalid base64-encoded value`.
- Code Mode `image()` / `generatedImage()` helpers require a `data:` scheme
  (parity with codex-rs code-mode) and reject bare paths and other schemes at
  the JavaScript boundary.
- Image-processing error classification also matches Codex invalid-`image_url`
  400s so the sampler can strip remaining images and retry once if needed.

## Install

This release provides a raw binary for **Apple Silicon macOS only**.

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/download/v0.1.220-open-grok.10/install.sh | bash
open-grok --version
```

The installer verifies the published SHA-256 digest and atomically installs
only `open-grok` to `${OPENGROK_HOME:-$HOME/.opengrok}/bin`. It does not replace
or create `grok` or `agent` commands, and it does not read or write `~/.grok`.
