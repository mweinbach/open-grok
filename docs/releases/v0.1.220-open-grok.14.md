# Open Grok v0.1.220-open-grok.14

Open Grok is Grok Build with ChatGPT Codex optimizations, packaged as the
independent `open-grok` command with isolated state under `OPENGROK_HOME`.

## Codex stream reliability

- Codex `web_search_call` items that arrive without an `action` payload — on
  any frame, including terminal `response.output_item.done` and
  `response.completed` outputs — now parse through a typed-boundary sentinel
  instead of failing the whole turn with a non-retryable
  ``missing field `action` `` serialization error. The sentinel is stripped
  from request bodies before replay, so fabricated actions never reach the
  provider, and it renders as a plain web-search label in transcripts.

## Subagent resilience

- Subagents that die on a terminal session error no longer discard their
  work. The result keeps the child's last assistant text and token usage,
  and the task tool's failure message now includes the partial output plus a
  `resume_from` handle, so the parent can continue the preserved child
  session instead of restarting from zero. Cancellations and spawn
  rejections are unchanged.

## Reasoning presentation

- Codex reasoning summaries stream and persist as separate markdown
  paragraphs. Each summary part opens with a bolded headline; previously
  parts were concatenated without a separator (live) or joined with a single
  newline (history), so consecutive headlines rendered on one run-on line.

## Stability and tests

- Resolving the external OTEL master switch no longer runs a full layered
  config load inside every `Config` deserialize. The resolution is memoized
  for the process lifetime, matching its startup-time activation semantics,
  which also fixes a debug-build stack overflow during per-turn auth-fact
  resolution.
- The persistent **Swarm mode** settings toggle announced in `.13` is fully
  wired: `[ui].swarm_mode` persists through the Settings UI and survives a
  config round-trip.
- Interactive folder-trust tests now key on the folder-trust method instead
  of "any gateway message", so legitimate `x.ai/models/update` catalog
  pushes no longer flake full-suite runs.

## Install

This release provides a raw binary for **Apple Silicon macOS only**.

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/download/v0.1.220-open-grok.14/install.sh | bash
open-grok --version
```

The installer verifies the published SHA-256 digest and atomically installs
only `open-grok` to the selected Open Grok bin directory under
`OPENGROK_HOME`. It does not replace or create `grok` or `agent` commands,
and it does not read or write `~/.grok`.
