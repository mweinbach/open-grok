# Open Grok v0.1.220-open-grok.16

Open Grok is Grok Build with ChatGPT Codex and Kimi provider support, packaged
as the independent `open-grok` command with isolated state under
`OPENGROK_HOME`.

This release makes web search a per-provider choice, brings X search to every
provider, and fixes a set of real-world bugs found in live sessions — most
notably plan approval failing to hold the model's turn in Code Mode.

## Web search source per provider

- New Settings → Models rows select the web search backend independently for
  xAI, Codex, Kimi Platform, and Kimi Code sessions, persisted under
  `[toolset.web_search_source]`.
- The xAI web search tool is client-executed (not a provider server tool), so
  it can serve any model: Kimi sessions now default to xAI web search when
  signed in, Codex keeps its native search by default with xAI and Perplexity
  as options, and Kimi falls back to none when neither backend is signed in.
- Choosing a non-native source for Codex suppresses the native hosted search
  declaration — only when the chosen source's credentials actually resolve,
  so a missing key never leaves a session without search.
- The legacy `[toolset.perplexity_web_search]` toggle keeps working as the
  Kimi default, and an explicitly configured web-search model still opts
  Codex into xAI search as before. Selections are read at session spawn and
  apply to new sessions without a restart.

## X search for every provider

- A new client-executed `x_search` tool, backed by the xAI API, gives Codex
  and Kimi sessions live X (Twitter) search. It registers only when xAI is
  signed in, can be disabled via the new X search setting
  (`[toolset.x_search]`), and xAI sessions keep the provider-hosted
  declaration so the two never coexist on one request.
- In Code Mode the tool ships as the first deferred nested tool: callable
  from cells via the `tools` object without bloating every exec description.

## Plan mode holds the turn in Code Mode

- Plan lifecycle tools are now direct-only in Code Mode, so `exit_plan_mode`
  parks the model's turn on the user's plan approval instead of running
  nested inside an `exec` cell whose yield timer handed the model turns to
  burn while the approval prompt was still open.
- `exec`/`wait` additionally hold yielded cells whenever a blocking user
  interaction (permission prompt, question, plan approval) is pending,
  re-waiting silently and preserving all cell output, so the same class of
  leak is closed for every interactive tool.

## Fixes

- Plugin hook commands referencing `${CLAUDE_PLUGIN_ROOT}` /
  `${CLAUDE_PLUGIN_DATA}` now always resolve to the plugin's own
  directories, even when the Open Grok process itself runs inside another
  agent's plugin environment that exports those names.
- Message-only `update_goal` calls against an absent or non-Active goal now
  reject with an accurate reply instead of acking success, so the model no
  longer retries doomed `completed: true` follow-ups.
- Unit tests are isolated from the live `OPENGROK_HOME`: test runs can no
  longer rotate away real logs, overwrite model caches, or create junk
  session directories.
- A debug-build stack overflow in the persist-ack test no longer aborts the
  test binary, and the model-catalog tests track the `grok-4.5` embedded
  default.

## Upgrade

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/latest/download/install.sh | bash
```

Or from a running session: `/update`.
