# Open Grok v0.1.220-open-grok.19

Open Grok is Grok Build with ChatGPT Codex, Kimi, and Fireworks AI provider
support, packaged as the independent `open-grok` command with isolated state
under `OPENGROK_HOME`.

This release keeps subagents out of the plan-mode lifecycle. Previously a
child agent (for example an `explore` audit subagent) could call
`exit_plan_mode`, and its approval dialog rendered on the main session's
view — indistinguishable from the main agent presenting its plan, often with
an empty plan file. Approving it exited only the child's plan mode while the
main agent kept running, which read as "a subagent left the main agent's
plan mode".

## Subagent plan-mode fix

- Subagent spawns strip the `enter_plan_mode` / `exit_plan_mode` tools under
  every capability mode (`read-only`, `read-write`, `execute`, `all`), so
  children can no longer drive the plan-mode lifecycle at all.
- Shell defense: a child session that still reaches `exit_plan_mode`
  (verbatim-mirror forks, sessions persisted before this release) executes
  the exit directly without the client approval round-trip, and a resumed
  child clears a stale `awaiting_plan_approval` flag instead of re-raising
  approval chrome on the parent's view.
- Pager defense: an `x.ai/exit_plan_mode` reverse-request from a child
  session is auto-approved instead of parking a plan-approval overlay on the
  parent agent's view. Plan approvals from background root sessions still
  land on their own view, unchanged.

## Upgrade

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/latest/download/install.sh | bash
```

Or from a running session: `/update`.
