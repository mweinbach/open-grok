# Open Grok v0.1.220-open-grok.12

Open Grok is Grok Build with ChatGPT Codex optimizations, packaged as the
independent `open-grok` command with isolated state under `OPENGROK_HOME`.

## Search and provider tooling

- Kimi Platform and Kimi Code can opt into a Perplexity raw Search API fallback
  when their provider profile lacks native web search. xAI and Codex continue
  to use their provider-native search paths, and the public
  `web_search(query, allowed_domains?)` contract is unchanged.
- The Perplexity key is stored only in owner-protected `auth.json` under the
  isolated `perplexity::api_key` scope. It never enters model credentials,
  `config.toml`, session persistence, or diagnostics.
- Settings can enable the fallback and update its key live. Queued Kimi sends
  pause until durable settings and every resident session have been rebound,
  with rollback and reconciliation on failure.

## Security and runtime reliability

- Compatible hardening from the pinned Grok Build 0.2.102 snapshot adds signed
  policy claim and rollback checks, exact-SHA remote plugin policy, narrower
  invalid-hook matching, owner-only secret files, and child-process network
  enforcement under the sandbox.
- xAI credentials remain restricted to exact trusted endpoints. Authentication
  refresh is single-flight, preventing concurrent turns from racing or
  exporting a token to an untrusted route.
- Session and resource JSONL writes are durable, notification acknowledgements
  persist correctly, and saved subagent results are bounded. MCP setup and
  managed configuration updates also recover more safely from interruption.
- In plan mode, `apply_patch` may add or update only the exact session
  `plan.md`. Malformed, mixed-target, delete, move, and symlink-detour patches
  fail closed before permissions.

## Terminal and interface

- `open-grok wrap` restores terminal state after clean exit, SIGTERM, or a
  terminated child. Clipboard diagnostics now explain unsupported terminals
  and unverified SSH clipboard paths more clearly.
- Dashboard peek and live-tail behavior are more stable, while login and MCP
  setup use single-flight state transitions that avoid duplicate work.
- Exit output includes a contextual resume hint. Queue and session-title polish,
  improved shortcut rendering, and the modular textarea editor make long
  interactive sessions more predictable.

## Install

This release provides a raw binary for **Apple Silicon macOS only**.

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/download/v0.1.220-open-grok.12/install.sh | bash
open-grok --version
```

The installer verifies the published SHA-256 digest and atomically installs
only `open-grok` to the selected Open Grok bin directory under
`OPENGROK_HOME`. It does not replace or create `grok` or `agent` commands,
and it does not read or write `~/.grok`.
