# Open Grok v0.1.220-open-grok.23

Open Grok is Grok Build with ChatGPT Codex, Kimi, and Fireworks AI provider
support, packaged as the independent `open-grok` command with isolated state
under `OPENGROK_HOME`.

This release makes `apply_patch` failures recoverable in one attempt instead
of fueling identical-retry loops — the failure mode real sessions kept
hitting when a hunk's context lines didn't match the file.

## Patch failures now point at the real file

When a hunk's expected lines can't be found, the error no longer just echoes
the patch back. It now appends the most similar region of the target file,
with line numbers and a similarity score:

```
Failed to find expected lines in Sources/ViewModel.swift:
    var showingRestPicker = false
    var showingExerciseEditor = false

Closest match in Sources/ViewModel.swift at lines 3-5 (similarity: 77.9%):
3	    var currentRestDuration: TimeInterval = 0
4	    var showingExerciseEditor = false
```

Hunks whose lines exist but occur *before* where the previous hunk matched
are called out explicitly — out-of-order hunks were previously
indistinguishable from wrong context. The codex-format first line and the
success output are byte-identical to before, so codex-trained models stay
on-distribution.

## All bad hunks in one round trip

Multi-file patches are validated hunk-by-hunk to the end instead of stopping
at the first failure. One response now reports every failing hunk, states
that nothing was applied (application stays atomic), and lists which hunks
were valid — so the model fixes everything at once rather than discovering
failures serially across retries.

## Prompt example matches the tool schema

The apply_patch system prompt showed a codex shell-command invocation
(`{"command":["apply_patch",...]}`) that does not exist in this harness. It
now shows the tool's real `{"patch":"*** Begin Patch..."}` schema.

## Upgrade

```sh
curl -fsSL https://github.com/mweinbach/open-grok/releases/latest/download/install.sh | bash
```

Or from a running session: `/update`.
